First-Person Arcade Room Setup Guide

Overview

This is a first-person movement and camera system for an arcade room scene.

Scripts:

* This script handles movement, crouching, sprinting, jumping and ground detection.

* PlayerCameraController.cs. This script handles mouse look, head bobbing and field of view adjustment.

### Scene Setup

1. Create Player Hierarchy

* Player (Root)

* Body (child handles rotation)

* CameraHolder (child)

* Main Camera (Camera component

2. Player (Root GameObject)

* Add these components:

Rigidbody

. Body Type: Dynamic

. Mass: 1

. Drag: 0

. Angular Drag: 0

. Gravity: enabled

. Constraints: Freeze Rotation X, Y Z

+ CapsuleCollider

. Radius: 0.3

. Height: 2

. Center: (0, 1 0)

* Add PlayerMovementController script

3. Body (Child of Player)

* Empty GameObject

* Position: (0, 0 0)

* This rotates horizontally for mouse look

4. CameraHolder (Child of Body)

* Empty GameObject

* Position: (0, 0.6 0)

* This rotates vertically for mouse look

5. Main Camera (Child of CameraHolder)

* Add Camera component

* Field of View: 60

* Add PlayerCameraController script

* Position: (0, 0 0)

* Rotation: (0, 0 0)

### Input System Setup

You need to set up Input Actions. If using the default Unity Input System create actions:

* Action Map: "Player"

* Move

. Value Type: Value

. Control Type: Vector2

. Use WASD keys or Gamepad Left Stick

* Sprint

. Value Type: Button

. Control Type: Button

. Use Left Shift key

* Crouch

. Value Type: Button

. Control Type: Button

. Use Left Ctrl key

* Jump

. Value Type: Button

. Control Type: Button

. Use Space key

* Max Look Up Angle: 90

* Max Look Down Angle: 90

### Ground Layer Setup

* Create a Layer called "Ground"

* Assign all ground/floor colliders to this layer

* In PlayerMovementController set Ground Layer to "Ground"

### Troubleshooting

* Player falls through ground?

Ensure ground colliders have colliders

+ Make sure "Ground" layer is set correctly

+ Check that player is, on the "Ground" layer or colliding with it

* Camera moves weirdly?

Verify camera is child of Body

+ Check that Body is child of Player

+ Ensure rotations are properly inherited

* No movement?

Verify Input Actions are set up

+ Check PlayerMovementController can find Rigidbody/Collider

+ Ensure Ground Layer is assigned

* Head bobbing too strong?

+ Lower Head Bob Amplitude

+ Lower Head Bob Frequency
